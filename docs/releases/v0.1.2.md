## Highlights

- Added `wait_kimi_task`, a dedicated terminal-wait tool that avoids high-frequency Codex polling.
- Increased the fallback polling suggestion from 1.5 seconds to 20 seconds.
- Preserved the existing `get_kimi_task` contract for quick snapshots and backward-compatible waits.

## Efficient async protocol

- `wait_kimi_task` waits 45 seconds by default and accepts an explicit wait from 1 to 300 seconds.
- Intermediate phase updates no longer end a long wait; the tool returns when the task becomes terminal or the wait expires.
- Active task reads return only compact status fields.
- Terminal task reads still return the complete task, attempt history, summary or error, and Git change receipt required for Codex review.

## Compatibility

- Existing task IDs and persisted task files remain compatible.
- `get_kimi_task`, `start_kimi_task`, `continue_kimi_task`, and `cancel_kimi_task` keep their existing names and responsibilities.
- No safety boundary changed: Codex still owns scope, Git review, testing, browser acceptance, and final verification.

## Verification

- 47 / 47 automated tests pass.
- Long-wait limits, bounded status-check counts, intermediate progress, compact active responses, and complete terminal receipts are covered.
- The five-tool bundled MCP server passes isolated-distribution and syntax checks.
